## Understanding the Delivery Workflow

### Order Status Progression

Your orders will progress through these statuses:

```
1. Pending
   ‚Üì (You accept the order)
2. Runner Accepted
   ‚Üì (You arrive at ATM)
3. Runner at ATM
   ‚Üì (You withdraw cash and generate OTP)
4. Cash Withdrawn / Pending Handoff
   ‚Üì (Customer provides OTP, you verify)
5. Completed ‚úÖ
```

### Visual Status Indicators

- **Gray Badge** = Pending (waiting for acceptance)
- **Blue Badge** = In Progress (Runner Accepted, At ATM, Pending Handoff)
- **Green Badge** = Completed (delivery successful)
- **Red Badge** = Cancelled (order cancelled)

## Complete Delivery Process

### Phase 1: Accepting an Order

**What You'll See:**
- Cash amount to withdraw (e.g., $100.00)
- Your earnings (e.g., $8.16)
- Delivery address
- Customer notes (if any)

**What To Do:**
1. Review the order details
2. Check the delivery location
3. Click **"Accept Order"** button
4. You'll be taken to the order detail page

**What Happens:**
- ‚úÖ Order status changes to "Runner Accepted"
- ‚úÖ Status badge turns blue
- ‚úÖ Customer is notified
- ‚úÖ Order removed from other runners' available list

---

### Phase 2: Going to the ATM

**Your Task:**
- Head to the nearest ATM to withdraw the requested cash amount

**What You'll See:**
- Order detail page with delivery steps
- Step 1: "Go to ATM" section
- Button: **"I've Arrived at ATM"**

**What To Do:**
1. Travel to an ATM location
2. When you arrive, click **"I've Arrived at ATM"**

**What Happens:**
- ‚úÖ Order status changes to "Runner at ATM"
- ‚úÖ Customer sees you've arrived at ATM
- ‚úÖ Customer can no longer cancel the order

---

### Phase 3: Withdrawing Cash

**Your Task:**
- Withdraw the exact cash amount shown in the order

**What You'll See:**
- Step 2: "Withdraw Cash" section
- Cash amount to withdraw
- Button: **"Cash Withdrawn - Generate OTP"**

**What To Do:**
1. Withdraw the cash from the ATM
2. Count and verify the amount
3. Click **"Cash Withdrawn - Generate OTP"**

**What Happens:**
- ‚úÖ Order status changes to "Pending Handoff"
- ‚úÖ A 6-digit OTP code is generated
- ‚úÖ OTP is sent to the customer
- ‚úÖ You can now proceed to meet the customer

**Important Notes:**
- The OTP is valid for 10 minutes
- Maximum 3 verification attempts allowed
- If OTP expires, you'll need to generate a new one

---

### Phase 4: Meeting the Customer

**Your Task:**
- Travel to the delivery address and meet the customer

**What You'll See:**
- Step 3: "Complete Delivery" section
- 6-digit OTP input field
- Button: **"Verify & Complete Delivery"**

**What To Do:**
1. Navigate to the delivery address
2. Meet the customer in person
3. Ask the customer for their 6-digit verification code
4. Enter the code in the OTP input field
5. Click **"Verify & Complete Delivery"**

**What Happens:**
- ‚úÖ System verifies the OTP code
- ‚úÖ If correct: Order status changes to "Completed"
- ‚úÖ Status badge turns green
- ‚úÖ You hand over the cash to the customer
- ‚úÖ Your earnings are added to your monthly total
- ‚úÖ Order marked as complete

**If OTP is Incorrect:**
- ‚ùå Error message displayed
- ‚ùå OTP field cleared
- ‚ùå You can try again (up to 3 attempts)
- ‚ùå After 3 failed attempts, new OTP must be generated

## Troubleshooting Common Issues

### Issue: Order Not Appearing in Available Orders

**Possible Causes:**
- You're not logged in with a runner account
- Order has already been accepted by another runner
- Order status is not "Pending"

**Solutions:**
- Verify you're logged in with runner credentials
- Check "My Deliveries" to see if you already accepted it
- Wait for new orders to be created
- Contact admin if issue persists

---

### Issue: Status Badge Not Updating

**This Issue Has Been Fixed! ‚úÖ**

**What Was Wrong:**
- Status badge was showing "Pending" even after acceptance
- Visual display didn't match actual order state

**What's Fixed:**
- Status badge now updates dynamically
- Colors change based on order status:
  - Gray = Pending
  - Blue = In Progress
  - Green = Completed
  - Red = Cancelled

**If You Still See Issues:**
- Clear your browser cache
- Log out and log back in
- Check browser console for errors
- Contact admin for support

---

### Issue: OTP Not Generating

**Possible Causes:**
- Order status is not "Runner at ATM"
- Network connection issue
- System error

**Solutions:**
- Ensure you clicked "I've Arrived at ATM" first
- Check your internet connection
- Try clicking "Cash Withdrawn - Generate OTP" again
- If problem persists, contact admin

---

### Issue: OTP Verification Failing

**Possible Causes:**
- Incorrect OTP code entered
- OTP has expired (10-minute limit)
- Maximum attempts exceeded (3 attempts)

**Solutions:**
- Double-check the code with the customer
- Ensure all 6 digits are entered correctly
- If expired, generate a new OTP
- If attempts exhausted, generate a new OTP
- Contact admin if customer didn't receive OTP

---

### Issue: Cannot Complete Delivery

**Possible Causes:**
- OTP code is incorrect
- OTP has expired
- Network connection issue

**Solutions:**
- Verify the 6-digit code with customer
- Check OTP hasn't expired (10 minutes)
- Ensure you have internet connection
- Try entering the code again
- Generate new OTP if needed
- Contact admin for assistance

## Safety & Security

### Cash Handling

- Always count cash before leaving the ATM
- Keep cash secure during transport
- Never leave cash unattended
- Count cash with customer during handoff

### Personal Safety

- Meet customers in well-lit, public areas when possible
- Trust your instincts - if something feels wrong, contact admin
- Keep your phone charged and accessible
- Share your location with a trusted contact

### OTP Security

- Never share OTP codes with anyone except the customer
- Verify customer identity before showing OTP
- Don't complete delivery without proper OTP verification
- Report any suspicious OTP requests to admin
